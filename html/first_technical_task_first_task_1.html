Once you have your first technical task assigned by CTO you can start working on it.

The simplest workflow would be:

  1. Change the JIRA status of the ticket you're going to work on to In Progress .
  2. Do git checkout of the master branch in the repository you're going to work with. Pull from remote.
  3. Create a new branch of the master branch. You can name it like: your_name/name_of_the_new_feature .
  4. Implement a new feature.
  5. Test your code, both manually and with tests, and make sure your code passes quality checks and is generally high-standard and readable for a reviewer.
  6. Push your branch to remote.
  7. Open a pull request on GitHub.
  8. Change the JIRA status of your ticket to Internal review and ping your reviewer.

Important note:

The scenario above describes the simplest possible workflow but please do note that workflows can differ drastically for different tasks and one needs to ultimately figure out what works best through experience. Please make sure you have read the documentation on task workflows , ticket statuses and typical process before starting your work, to understand how workflows may differ for more specific scenarios; but again, you should be adaptable.
